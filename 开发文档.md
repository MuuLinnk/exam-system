# 学生考试系统 - 开发设计文档

---

## 一、系统概述

一个**中文界面**的网页版在线考试系统。学生在浏览器里打开网址，注册账号后登录、参加考试、提交答卷、查看成绩。

**核心流程：**

```
注册 → 登录 → 查看考试列表 → 进入考试（计时开始）→ 答选择题/填空题/简答题 → 提交答卷 → 查看成绩
```

### 数据库连接配置

Spring Boot 配置文件 `application.properties`：

```properties
# MySQL 连接
spring.datasource.url=jdbc:mysql://localhost:3306/exam_system?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf-8
spring.datasource.username=root
spring.datasource.password=666666
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

# JPA 自动建表
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.MySQLDialect
```

> **启动前需手动创建数据库**：用 Navicat 或命令行执行 `CREATE DATABASE exam_system DEFAULT CHARACTER SET utf8mb4;`
> 
> 表由 JPA 自动创建，不需要手动建表。

## 三、数据库设计（详细）

### 3.1 学生表 `student`

| 字段名 | 类型 | 长度 | 说明 | 约束 |
|--------|------|------|------|------|
| id | BIGINT | - | 主键 | 自增 |
| student_no | VARCHAR | 20 | 学号 | 唯一，不能为空 |
| name | VARCHAR | 50 | 姓名 | 不能为空 |
| password | VARCHAR | 100 | 密码（加密存储） | 不能为空 |
| class_name | VARCHAR | 50 | 班级 | 可为空 |
| create_time | DATETIME | - | 注册时间 | 自动生成 |

### 3.2 题目表 `question`

| 字段名 | 类型 | 长度 | 说明 | 约束 |
|--------|------|------|------|------|
| id | BIGINT | - | 主键 | 自增 |
| content | TEXT | - | 题目内容 | 不能为空 |
| type | VARCHAR | 10 | 题目类型 | CHOICE / BLANK / ESSAY |
| option_a | VARCHAR | 200 | 选项A | 选择题用 |
| option_b | VARCHAR | 200 | 选项B | 选择题用 |
| option_c | VARCHAR | 200 | 选项C | 选择题用 |
| option_d | VARCHAR | 200 | 选项D | 选择题用 |
| answer | VARCHAR | 500 | 正确答案 | 不能为空 |
| score | INT | - | 分值 | 默认 5 分 |
| difficulty | VARCHAR | 10 | 难度 | EASY / MEDIUM / HARD |

**题目类型说明：**
- `CHOICE`（选择题）：存 A/B/C/D，有四个选项字段
- `BLANK`（填空题）：存文字答案，选项字段为空
- `ESSAY`（简答题）：存参考答案，选项字段为空

### 3.3 试卷表 `exam_paper`

| 字段名 | 类型 | 长度 | 说明 | 约束 |
|--------|------|------|------|------|
| id | BIGINT | - | 主键 | 自增 |
| title | VARCHAR | 100 | 试卷名称 | 不能为空 |
| duration | INT | - | 考试时长（分钟） | 不能为空 |
| total_score | INT | - | 试卷总分 | 不能为空 |
| create_time | DATETIME | - | 创建时间 | 自动生成 |

### 3.4 试卷-题目关联表 `paper_question`

| 字段名 | 类型 | 长度 | 说明 | 约束 |
|--------|------|------|------|------|
| id | BIGINT | - | 主键 | 自增 |
| paper_id | BIGINT | - | 试卷ID | 外键 |
| question_id | BIGINT | - | 题目ID | 外键 |
| sort_order | INT | - | 题目序号 | 默认 1 |

> 一张试卷包含多道题，一道题可以出现在多张试卷里，所以用这张中间表关联。

### 3.5 考试记录表 `exam_record`

| 字段名 | 类型 | 长度 | 说明 | 约束 |
|--------|------|------|------|------|
| id | BIGINT | - | 主键 | 自增 |
| student_id | BIGINT | - | 学生ID | 外键 |
| paper_id | BIGINT | - | 试卷ID | 外键 |
| answers | TEXT | - | 学生答案（JSON） | 不能为空 |
| score | INT | - | 得分 | 默认 0 |
| status | VARCHAR | 10 | 状态 | DOING / SUBMITTED |
| start_time | DATETIME | - | 开始考试时间 | - |
| submit_time | DATETIME | - | 提交时间 | 可为空 |

**answers 字段 JSON 格式示例：**
```json
[
  { "questionId": 1, "type": "CHOICE", "answer": "A" },
  { "questionId": 2, "type": "BLANK", "answer": "北京" },
  { "questionId": 3, "type": "ESSAY", "answer": "因为...所以..." }
]
```

## 四、API 接口设计

> 接口 = 后端提供的 URL 地址。前端访问这些地址来获取数据或提交数据。
> 
> 返回格式统一为：`{ "code": 200, "message": "成功", "data": {...} }`

### 4.1 注册

```
POST /api/student/register
```

**前端发过去：**
```json
{
  "studentNo": "2024001",
  "name": "张三",
  "password": "123456",
  "className": "计科一班"
}
```

**后端返回：**
```json
{
  "code": 200,
  "message": "注册成功",
  "data": null
}
```

**可能的错误：**
- `400`：学号已存在
- `400`：学号或密码为空

### 4.2 登录

```
POST /api/student/login
```

**前端发过去：**
```json
{
  "studentNo": "2024001",
  "password": "123456"
}
```

**后端返回：**
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "xxxxxxxxxxxxx",
    "studentName": "张三"
  }
}
```

> token 是一个字符串"通行证"，后续所有请求都要在请求头里带上它。

### 4.3 获取考试列表

```
GET /api/exam/list
请求头：token: xxxxxxxxxxxxx
```

**后端返回：**
```json
{
  "code": 200,
  "data": [
    {
      "paperId": 1,
      "title": "Java 基础测验",
      "duration": 60,
      "totalScore": 100,
      "status": "NOT_STARTED"
    },
    {
      "paperId": 2,
      "title": "数据结构期中考试",
      "duration": 120,
      "totalScore": 100,
      "status": "SUBMITTED"
    }
  ]
}
```

**status 说明：**
- `NOT_STARTED`：还没开始考
- `DOING`：考试进行中
- `SUBMITTED`：已提交

### 4.4 开始考试（获取试卷内容）

```
GET /api/exam/start/{paperId}
请求头：token: xxxxxxxxxxxxx
```

**后端返回：**
```json
{
  "code": 200,
  "data": {
    "recordId": 1,
    "paperTitle": "Java 基础测验",
    "duration": 60,
    "totalScore": 100,
    "questions": [
      {
        "questionId": 1,
        "type": "CHOICE",
        "content": "Java 中 int 占几个字节？",
        "options": { "A": "2", "B": "4", "C": "8", "D": "1" },
        "score": 5
      },
      {
        "questionId": 2,
        "type": "BLANK",
        "content": "HTML 的中文全称是______。",
        "score": 5
      },
      {
        "questionId": 3,
        "type": "ESSAY",
        "content": "请简述面向对象的三大特性。",
        "score": 15
      }
    ],
    "remainingSeconds": 3600
  }
}
```

> 注意：这里**不返回正确答案**，只返回题目内容。正确答案只在提交后才返回。
>
> recordId 是这次考试记录的 ID，提交答案时要用。

### 4.5 提交答案

```
POST /api/exam/submit
请求头：token: xxxxxxxxxxxxx
```

**前端发过去：**
```json
{
  "recordId": 1,
  "answers": [
    { "questionId": 1, "answer": "B" },
    { "questionId": 2, "answer": "超文本标记语言" },
    { "questionId": 3, "answer": "封装、继承、多态。" }
  ]
}
```

**后端返回：**
```json
{
  "code": 200,
  "message": "提交成功",
  "data": {
    "totalScore": 100,
    "objectiveScore": 10,
    "details": [
      {
        "questionId": 1,
        "yourAnswer": "B",
        "correctAnswer": "B",
        "correct": true,
        "score": 5
      },
      {
        "questionId": 2,
        "yourAnswer": "超文本标记语言",
        "correctAnswer": "超文本标记语言",
        "correct": true,
        "score": 5
      },
      {
        "questionId": 3,
        "yourAnswer": "封装、继承、多态。",
        "correctAnswer": "封装、继承、多态",
        "correct": null,
        "score": 0,
        "remark": "待批改"
      }
    ]
  }
}
```

**评分逻辑（写在后端 Service 里）：**
- 选择题：学生答案 == 正确答案 → 得分
- 填空题：学生答案 == 正确答案（忽略首尾空格）→ 得分
- 简答题：不自动判分，`correct` 返回 `null`，标记"待批改"

### 4.6 查看考试记录详情

```
GET /api/exam/record/{recordId}
请求头：token: xxxxxxxxxxxxx
```

返回内容和 4.5 提交后返回的 `data` 结构相同，用于学生回看已提交的考试。

**答题页逻辑：**
1. 进入页面调用 `GET /api/exam/start/{paperId}`
2. 拿到试卷数据后，开始倒计时
3. 所有题目在一个长页面上（滚动浏览）
4. 学生每选/填一个答案，自动暂存在前端内存里
5. 时间到 → 自动提交 → 跳转到成绩页
6. 点"确认提交"→ 弹窗二次确认 → 提交 → 跳转成绩页

**三个题型组件：**
- `ChoiceQuestion.vue`：接收 `{ content, options, score }`，用 `<el-radio-group>` 渲染单选项
- `BlankQuestion.vue`：接收 `{ content, score }`，用 `<el-input>` 渲染输入框
- `EssayQuestion.vue`：接收 `{ content, score }`，用 `<el-input type="textarea">` 渲染大文本框

## 六、安全设计

| 措施 | 说明 |
|------|------|
| 密码加密 | 用 BCrypt 加密存数据库，不存明文 |
| Token 校验 | 每个请求都验证 token，防止未登录访问 |
| 考试防作弊（基础） | 后端记录考试开始时间，防止超时后继续提交 |
